# Evaluation Report

## 1. Evaluation Goal

This evaluation checks whether ClipPilot-Agent behaves as a practical Multi-Agent Editing Workflow Harness: fewer invalid exports, clearer traceability, and stronger gate behavior. It is not a public video summarization benchmark.

## 2. Dataset

The current framework defines five starter cases in `eval/cases.yaml`, including good subtitles, bad ASR, degenerate single-segment behavior, policy overflow, and media validation. Video paths are environment-variable based and local-only.

Human review status: `pending`.

## 3. Compared Systems

- Rule Baseline
- Single-Agent Baseline
- Multi-Agent Harness

## 4. Human Review Protocol

`eval/outputs/human_review_sheet.csv` is blinded and randomized. It does not expose `system_name`, and all human score fields remain empty until a person reviews the artifacts.

## 5. Main Results

Automated result files are generated by:

```powershell
py -3.12 eval/run_eval.py
py -3.12 eval/build_human_review_sheet.py
py -3.12 eval/aggregate_results.py
```

Human-dependent metrics such as Useful Completion Rate remain pending until review decisions are filled.

## 6. Gate Behavior

The current cases cover transcript blocking, coverage blocking, policy blocking, media blocking, and normal export-or-block behavior. These cases are primarily used to verify gate behavior, not to prove content quality.

## 7. Success Cases

See `docs/cases/success_case_good_vtt.md`.

## 8. Failure Cases

See `docs/cases/failure_case_physics_asr.md`. That case shows media-valid output can still fail task coverage and remain not production-ready.

## 9. Conclusions

Current evidence supports the engineering claim that ClipPilot-Agent is a runnable, testable workflow harness with trace, replay, repair, and validation. It does not yet prove that the Multi-Agent output is better than a single-agent or rule baseline in human content quality.

## 10. Limitations

The sample size is small, human review is pending, and no public benchmark superiority is claimed.
